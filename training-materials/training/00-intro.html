Introduction

ODE is a system that can be used to design natural language output for intelligent agents (such as robots and non-player characters in computer games) in the form of rules. A rule consists of a name, a description, a left-hand side (LHS) and a right-hand side (RHS). The LHS specifies a number of conditions that must be fulfilled for the rule to apply. The corresponding RHS specifies the natural language output that should be produced by the agent if and when the rule applies.

As an example, consider the following rule: 

// @firstEncounter
// Agent meets someone for the first time.
<Encounter>first -> "Hi! Nice to meet you."

The name of this rule is firstEncounter; its description says that it covers situations in which an agent meets someone for the first time. The LHS of this rule consists of a single condition. Conditions are pairs of features and values. In the example above, Encounter is a feature and first is its value. The RHS of this rule consists of a single output string (Hi! Nice to meet you.). Output strings can – but do not have to – consist of multiple sentences.

To make conversations feel as natural as possible it is important for agents to be able to express the same content in many different ways. Rules support this by allowing you to specify any number of alternative verbalizations for the content to deliver. Every time a given rule applies, the agent will randomly choose and produce one of these alternative verbalizations. As a result, the more alternatives there are to choose from, the less likely the agent is to repeat itself.

Tasks

To become familiar with ODE's features for working with rules you will be performing a number of basic tasks. Each task consists of two parts: an introductory text providing the information you will need to complete the task, and a description of the task itself. Tasks are to be completed in the order in which they are presented, but you are allowed to go back to previous tasks to re-read the information they provide if necessary. If you find that you are having trouble understanding the description of a given task, or run into problems while performing a given task, please let the experimenter know.
